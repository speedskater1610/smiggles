# Filesystem

The filesystem stores file names and content, folder structure, and ownership metadata. The filesystem lives in a reserved disk region. It keeps two copies (slot A and slot B). Each slot's header has an identity marker, a version number, a generation counter, and a checksum. On startup, the filesystem code reads both slot headers/images, validates the checksums and versions, and chooses the newest valid one. If one slot is damaged, it automatically recovers from the other one. When saving, the current in-memory state is serialized into a fixed image structure, the checksum is computed, and the image is written to the inactive slot (with a new generation number going into the header). Each save alternates slots. This is to lower the risk of losing all the data if something happens mid-write.
